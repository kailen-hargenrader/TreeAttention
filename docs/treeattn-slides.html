Tree Attention

vs. Standard (Softmax) Attention

A walkthrough of how this repo's hierarchical attention kernels compute attention weights, where they diverge from the standard formulation, and what that costs.

Use → / Space to advance, ← to go back.

Standard attention, in one line

Given $N$ key/value pairs, each query $q$ attends to every value:

$$ \mathrm{Attn}(q, K, V) = \mathrm{softmax}\!\left(\frac{q K^\top}{\sqrt{d}}\right) V $$
  • Logits are computed for all $N$ keys, then normalized globally by softmax.
  • Per query: $O(N)$ dot products, $O(N)$ memory for weights.
  • FlashAttention-style kernels stream this in tiles to avoid materializing the $L \times N$ weight matrix, but the algorithm is unchanged.

Tree attention's setup

  • $N$ values sit at the leaves of a complete binary tree.
  • $K = N-1$ internal-node keys sit at branch points.
  • For each query, the attention weight on a leaf is the probability of walking the tree from root to that leaf, where each branch is decided by a sigmoid of $\langle q, k_{\text{node}}\rangle$.

$N=8$ leaves, $K=7$ internal keys, depth $\log_2 N = 3$.

The weight on one leaf is a path product

For leaf $i$, let $\text{path}(i) = (n_0, n_1, \dots, n_{L-1})$ be the internal-node keys on its root-to-leaf path, and $b_\ell \in \{0, 1\}$ the bit telling us whether the path goes left (0) or right (1) at level $\ell$. Define

$$ s_\ell \;=\; \langle q,\, k_{n_\ell}\rangle, \qquad p_\ell \;=\; \begin{cases} \sigma(-s_\ell) & b_\ell = 0 \;(\text{left}) \\ \sigma(+s_\ell) & b_\ell = 1 \;(\text{right}) \end{cases} $$

Then the attention weight on leaf $i$ is the product

$$ w_i \;=\; \prod_{\ell=0}^{L-1} p_\ell \;=\; \exp\!\Bigg(\sum_{\ell=0}^{L-1} \log \sigma\big((2b_\ell - 1)\, s_\ell\big)\Bigg). $$

This is exactly what tree_attention_deterministic_weights in the JAX kernel computes, level by level inside a lax.scan.

Weights sum to 1 — for free

At every internal node, $\sigma(-s) + \sigma(+s) = 1$. The two children of any node split the parent's mass exactly. So:

$$ \sum_{i=1}^{N} w_i \;=\; 1 \quad\text{by construction.} $$
  • Standard attention reaches $\sum w_i = 1$ via a global softmax — a single nonlocal normalization.
  • Tree attention reaches it via $\log_2 N$ local sigmoid decisions. No global normalization, no cross-key reduction.

This is the structural reason tree attention can be sampled cheaply (next slide).

Deterministic mode: enumerate all $N$ paths

Run the level-by-level recursion without branching — accumulate log-probs over a tensor of shape $L \times B \times H \times P$, doubling $P$ each level until $P = N$:

log_path_probs = zeros(L, B, H, 1)
for d in range(log_N):
    k_d  = k[(1< l b h n")
    logp = stack([logsigmoid(dots), logsigmoid(-dots)])  # left / right
    log_path_probs = log_path_probs[..., None] + logp    # broadcast & double
    log_path_probs = rearrange("l b h p two -> l b h (p two)")

attn_weights = exp(log_path_probs)                        # (L, B, H, N)
output       = einsum(attn_weights, v, "l b h n, n b h d -> l b h d")

Same answer as exhaustively walking every leaf. Cost: $\Theta(L\,N)$ FLOPs and a dense $(L, N)$ weight matrix — same asymptotics as standard attention, no savings.

Stochastic mode: Gumbel-max walk

Instead of evaluating every leaf, sample $S$ root-to-leaf paths per query. At each level, draw the branch with Gumbel-max:

$$ b \;=\; \arg\max\Big(\log p_\text{left} + g_0,\; \log p_\text{right} + g_1\Big), \quad g_j \sim \mathrm{Gumbel}(0,1). $$
  • $\log_2 N$ levels × $S$ paths × one dot-product per level = $O(S \log N)$ work and memory per query.
  • Approximates exact tree-attention output; bias controlled by $S$.

This is the regime where tree attention is supposed to beat standard attention asymptotically. (See the bench plot — fwd time grows much more slowly than $L$ in stochastic mode.)

Causal masking is harder than in standard attention

Standard

Set logits with $i > t$ to $-\infty$ before the softmax. One mask, done.

Tree

Truncating leaves can leave a partial subtree at the boundary whose mass would no longer sum to 1. Need a two-pass correction:

  1. Bottom-up: walk the boundary path from leaves up, computing the corrected log-mass of each ancestor (mixing on-path and off-path subtrees).
  2. Top-down: re-sample / re-evaluate using the corrected masses on the boundary path; off-boundary nodes use the regular $\sigma(\pm s)$.
# pseudo-pseudocode (per query t)
boundary = path_to(t)            # log_N internal nodes
log_mass[depth-1..0] from leaves up:
    L, R = subtrees of node at depth d
    if path goes left at d:
        on  = mass(L); off = -inf
    else:
        on  = mass(R); off = log(N_L)
    log_mass[d] = logsumexp(
        on  + logσ(±s_d),
        off + logσ(∓s_d))

then sample top-down,
using corrected sigmoids on boundary
and raw sigmoids elsewhere.

Implementation: _sample_paths_causal. The hybrid mode evaluates the first $S$ queries deterministically and only samples the rest.

The promise: stochastic-mode tree attention scales sub-linearly in $N$. The catch: deterministic-mode tree attention has the same asymptotics as standard attention but loses the FlashAttention-style streaming.

What this looks like in the harness

  • flash_attn_2 / sdpa_efficient: the streaming softmax baseline; sub-millisecond at 16k, MiB-scale memory.
  • treeattn_torch (deterministic): materializes the $(L, N)$ weight matrix and saves it for backward — that's the multi-GiB fwd_saved_mem_mb column you saw, and a $\Theta(L\,N)$ backward that explodes by 32k.
  • treeattn_jax (deterministic): forward uses a memory-light scan, but reverse-mode AD through vmap-of-scan materializes a $[B, H, L, N{-}1, D]$ transient at backward time. OOMs around 8k–16k even though forward saves "only" ~1 GiB.
  • Neither current implementation realizes the asymptotic stochastic-mode promise without further work (a custom VJP, blocked deterministic kernel, or jax.checkpoint on the scan).

One-slide summary

  • Standard attention: one global softmax over $N$ logits.
  • Tree attention: $\log_2 N$ local sigmoids along a binary path. Weights sum to 1 by construction, no normalization needed.
  • Deterministic mode = exhaustive tree walk = same asymptotics as standard attention.
  • Stochastic mode = Gumbel-max sample of $S$ paths = $O(S \log N)$ — the actual win.
  • Causal masking is non-trivial: needs a boundary-path log-mass correction.
  • Backward is the bottleneck in the current implementations; both kernels save (or re-derive) tensors that grow with $L \cdot N$.
